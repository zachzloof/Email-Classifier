# MailAI

An AI-powered email classifier for Outlook and Microsoft 365. MailAI connects to your mailbox via Microsoft Graph, classifies each email by category and priority using GPT-4.1-mini, and surfaces a clean inbox with one-line summaries and concrete suggested actions — packaged as a standalone Windows desktop app.

## Features

- **AI classification** — every email is assigned a category, a priority level, a one-line summary, and a specific suggested action, all generated by GPT-4.1-mini with structured JSON output
- **Custom categories** — define your own categories with plain-English descriptions; the descriptions are fed directly into the AI prompt so the quality of your description determines the quality of classification
- **Configurable sync** — choose how far back to pull (1–365 days) and how many emails per batch (1–100); already-classified emails are skipped automatically so repeated syncs progress through older mail
- **Delete with sync** — deleting an email in MailAI moves it to Outlook's Deleted Items and tombstones it locally so it never reappears on future syncs
- **Reply from the app** — compose and send replies directly without leaving MailAI
- **Filtering and sorting** — filter by category or priority; sort by date (newest/oldest) or priority (highest/lowest)
- **Try It** — paste any email subject and body into the classifier to preview how it would be categorised with your current settings, without triggering a full sync
- **Per-user storage** — each Microsoft account gets its own local SQLite database; multiple users can share one installation
- **OpenAI key management** — enter your API key directly in Settings; it persists across sessions and app updates
- **Amber warning banner** — the app warns you on every page if no OpenAI key is configured
- **Reset tools** — independently reset the email database or restore default categories (Work / Personal / Spam), with warnings about orphaned category labels
- **Dark mode** — follows the system theme
- **Windows installer** — ships as a standard NSIS `.exe` that installs to Program Files with a Start Menu shortcut and uninstaller

## How It Works

1. You sign in with your Microsoft account via a browser popup (PKCE flow — no client secret is stored)
2. On sync, MailAI fetches emails from your Outlook inbox via Microsoft Graph and sends each one to GPT-4.1-mini
3. The model classifies each email against your categories and returns a structured JSON response (category, priority, summary, suggested action)
4. Results are stored in a local SQLite database — emails already in the database are skipped on future syncs
5. The inbox view reads from the local database; no email data is fetched from Outlook on page load

The classifier prompt explicitly instructs the model to treat email content as data only and not follow any instructions found inside an email (prompt injection protection).

## Azure App Registration

You need to register MailAI with Azure so that users can sign in with their Microsoft account.

1. Go to [portal.azure.com](https://portal.azure.com) → **App registrations** → **New registration**
2. Give it a name (e.g. `MailAI`) and select **Personal Microsoft accounts only** under Supported account types
3. Leave the redirect URI blank for now and click **Register**
4. Under **Authentication** in the left sidebar:
   - Click **Add a platform** → **Mobile and desktop applications**
   - Add `http://localhost:3001/auth/callback` as a redirect URI and click **Configure**
   - Scroll to **Advanced settings** and set **Allow public client flows** to **Yes**
   - Click **Save**
5. Under **API permissions**:
   - Click **Add a permission** → **Microsoft Graph** → **Delegated permissions**
   - Add: `Mail.ReadWrite`, `Mail.Send`, `User.Read`, `offline_access`
   - Click **Add permissions**
6. Copy the **Application (client) ID** from the Overview page

> **Why Mobile and desktop applications?** This platform type tells Azure the app is a public client — it does not hold a client secret. The `Allow public client flows` toggle and the platform type together are both required; enabling only one is not sufficient.

The client ID is bundled in [api/index.js](api/index.js#L37). If you use your own registration, update that line.

## Development Setup

Install all dependencies from the project root (covers the API, Electron, and dev tooling — the UI has its own `package.json` managed via the root scripts):

```bash
npm install
```

Create a `.env` file at the project root for your OpenAI key:

```env
OPENAI_API_KEY=sk-...
```

Start both the API server and the Vite dev server concurrently:

```bash
npm run dev
```

| Service | URL |
|---|---|
| Frontend (Vite) | http://localhost:5173 |
| API (Express) | http://localhost:3001 |

In dev mode the frontend is served by Vite directly, so the Express static file serving is not used. The database files are written to `api/data/` and `api/tokens.db` — both are gitignored.

### Available scripts

| Script | Description |
|---|---|
| `npm run dev` | Start API + Vite dev servers concurrently |
| `npm run dev:api` | Start the Express API server only |
| `npm run dev:ui` | Start the Vite dev server only |
| `npm run build` | Build the Vue frontend and produce the Windows installer |
| `npm run build:ui` | Build the Vue frontend only (outputs to `ui/dist/`) |
| `npm start` | Launch the Electron app directly (requires a prior `build:ui`) |

## Building the Installer

Building requires either **Windows Developer Mode** enabled (`Settings → System → For Developers → Developer Mode`) or an **admin terminal**, because electron-builder extracts archives that contain symlinks and Windows restricts symlink creation by default.

```bash
npm run build
```

This will:
1. Build the Vue frontend with Vite into `ui/dist/`
2. Rebuild the `sqlite3` native module for the target Electron version
3. Package everything into `release/win-unpacked/` (portable, runnable directly)
4. Compile the NSIS installer to `release/MailAI Setup <version>.exe`

The installer is self-contained. Users run it, follow the setup wizard, and launch from the Start Menu shortcut. No Node.js, npm, or anything else is required on their machine.

> **Code signing:** The app is not code-signed. Windows SmartScreen will show a "More info → Run anyway" prompt on first install. Windows Smart App Control (found on fresh Windows 11 installs) is more aggressive and has no bypass — affected users must either turn SAC off in Windows Security settings or obtain a code signing certificate. For public distribution to unfamiliar users, a certificate from a CA such as Sectigo or DigiCert (~£200–400/year) removes both prompts entirely.

## Releasing a New Version

1. Bump `"version"` in [package.json](package.json) (e.g. `1.0.0` → `1.0.1`)
2. Run `npm run build` from an admin terminal
3. Distribute `release/MailAI Setup <version>.exe`

Installing a newer version over an existing one automatically uninstalls the previous version first. User data stored in `%AppData%\MailAI\` is never touched by the installer or uninstaller.

## First Launch

1. Open MailAI and click **Sign in with Microsoft**
2. A browser window opens — sign in and approve the requested permissions
3. You are redirected back to the app
4. Go to **Settings** and enter your OpenAI API key under **OpenAI API Key**
5. Go to **Inbox** and click **Sync** to pull and classify your emails

Token refresh happens silently in the background using the `offline_access` scope — you should not need to sign in again unless you explicitly log out.

## Using the App

### Syncing emails

Click **Sync** in the inbox toolbar to open the sync modal. You can configure:

- **Time range** — how far back to look (presets: 7 / 14 / 28 / 56 days, or enter a custom number up to 365)
- **Limit** — maximum number of new emails to classify per sync (presets: 25 / 50 / 100, max 100)

Emails already in the local database are skipped and do not count toward the limit. If the sync adds exactly as many emails as your limit, a hint appears suggesting you sync again to continue progressing through older mail.

### Categories

Categories live in **Settings**. Each has a name and a description. The description is included verbatim in the AI prompt — write it in plain English describing what kinds of emails should land there. More specific descriptions produce more accurate results.

Default categories (Work, Personal, Spam) can be edited or deleted just like any custom category. Categories can be restored to defaults at any time via **Settings → Reset → Reset Categories**.

> **Note:** Resetting categories does not re-classify existing emails. Emails classified under a deleted or renamed category will keep their old label in the inbox.

### Deleting emails

Deleting an email from MailAI does two things simultaneously:

1. Moves the email to **Deleted Items** in your Outlook mailbox via the Microsoft Graph API
2. Records the email's ID as a tombstone locally so it never reappears on future syncs

### Replying to emails

Open an email and use the reply form to compose and send a reply. The reply is sent via Microsoft Graph and appears in your Sent Items in Outlook.

### Try It (classifier sandbox)

Accessible from **Settings → Try It**. Paste a subject and body, click Classify, and see exactly how the model would label that content with your current categories and descriptions — useful for tuning your category descriptions without running a full sync.

## Data & Privacy

| What | Where |
|---|---|
| Classified emails, categories | `%AppData%\MailAI\data\<account>.db` (installed) or `api/data/<account>.db` (dev) |
| Auth tokens, OpenAI key | `%AppData%\MailAI\tokens.db` (installed) or `api/tokens.db` (dev) |

- All data is stored locally; nothing is sent to any server other than Microsoft Graph (for email access) and OpenAI (for classification)
- Email subject and body content is sent to the OpenAI API — review [OpenAI's data usage policy](https://openai.com/policies/api-data-usage-policies) if this is a concern for sensitive mailboxes
- Tokens are persisted using MSAL's serialisable cache — they are stored as-is in `tokens.db` and are not additionally encrypted

### Key architectural notes

- **Per-user databases** — `db.js` derives a filename from the signed-in account's email address. If two different Microsoft accounts use the app on the same machine, they each get a separate `.db` file with independent email and category data.
- **Static file serving** — in Electron (`SERVE_STATIC=true`), Express serves `ui/dist/` and catches all unmatched routes for Vue Router's history mode. In dev mode, Vite handles this instead.
- **Top-level await** — `api/index.js` uses top-level `await` to initialise the database before any routes are registered. This is valid because the file uses ES module syntax (`type: "module"` in `package.json`).
- **Native module** — `sqlite3` is a native Node.js addon. electron-builder rebuilds it for the target Electron Node.js version during `npm install` (via `postinstall`) and unpacks it from the asar archive at runtime (via `asarUnpack`).

## Troubleshooting

**The app won't start / crashes silently**
A startup error dialog should appear with a stack trace. The most common cause is a route syntax incompatibility with Express 5 — check that any wildcard routes use regex syntax (e.g. `/.*/`) rather than the Express 4 `'*'` string.

**"Auth failed: invalid_client" / AADSTS70002**
Azure is treating the app as a confidential client. In your Azure app registration under **Authentication**:
- Confirm the redirect URI `http://localhost:3001/auth/callback` is listed under **Mobile and desktop applications** (not Web)
- Confirm **Allow public client flows** is set to **Yes** and saved

**"Auth failed: redirect_uri_mismatch"**
The redirect URI in Azure does not exactly match `http://localhost:3001/auth/callback`. Check for trailing slashes or `https` vs `http`.

**Windows Smart App Control blocks the installer with no bypass option**
Smart App Control (found on fresh Windows 11 installs) blocks apps with no cloud reputation and provides no "Run anyway" option. Options:
1. Turn SAC off in **Windows Security → App & browser control → Smart App Control** (cannot be re-enabled without resetting Windows)
2. Obtain a code signing certificate

**SmartScreen shows "Windows protected your PC"**
Click **More info** → **Run anyway**. This is expected for unsigned apps and only appears once per installation.

**The installed app and the dev server conflict**
Both use port 3001. Do not run `npm run dev:api` or `npm run dev` while the installed app is open, and vice versa.

**Emails are not appearing after sync**
- Check that your OpenAI key is configured in Settings (a banner appears if it isn't)
- If syncing reports `added: 0`, all emails in the selected time window may already be in the local database — try extending the time range or increasing the limit

**Reinstalling wipes my emails**
It won't. The installer and uninstaller do not touch `%AppData%\MailAI\`. Your data persists across reinstalls and version upgrades.
